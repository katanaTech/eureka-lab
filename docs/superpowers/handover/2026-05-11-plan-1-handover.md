## Gotchas the previous session discovered

1. **Tailwind v4, NOT v3.** The project has no `tailwind.config.ts`. Tailwind config lives in `apps/web/src/app/globals.css` under `@theme inline { … }`. When adding fonts/keyframes/colors, edit globals.css, not a config file.

2. **Salvage commit needed a follow-up `fix(types)`.** The original Task 0.3 salvage restored `packages/shared-types/src/phase16.types.ts` but didn't update `packages/shared-types/src/index.ts` to re-export from it. API tsc was breaking with 24 missing-export errors. Fix `a631239` added `export * from './phase16.types';` to index.ts. The current `gameplay.types.ts` (renamed in 1.4) is re-exported via the same pattern.

3. **Pre-flight commit initially contained Phase-16-touched files that broke the revert.** Original pre-flight commit `4b60902` included `planning/sprint-p16.md`, `planning/blockers.md`, `docs/context/asset-licenses.md` — all Phase 16 artifacts. When Task 0.2 tried to revert `58c9f25`, those files caused modify/delete conflicts. Fixed by amending the pre-flight commit (`ea666fd`) to include ONLY `docs/superpowers/*` paths. If a future task touches Phase-16-only files, expect similar issues.

4. **`useAuthStore.user` is `UserProfile` (from `@eureka-lab/shared-types`), not Firebase `User`.** `displayName` and `email` are non-nullable strings on `UserProfile`. The `useGame()` adapter's `?? 'Hero'` / `?? ''` fallbacks are defensive but never trigger.

5. **`FantasyCharacter` backend ↔ `Character` frontend rename.** Backend: `{ name, class, classColorHsl, weaponName, createdAt }`. Frontend: `{ name, class, color, weaponName }`. The `characterApi` in `apps/web/src/lib/api-client.ts` maps both directions transparently (`color ↔ classColorHsl`). When porting the `/character` page (Task 2.5), the page reads/writes `Character` (frontend shape) — the api-client handles the rest.

6. **Reference project's `useGame()` API matches our adapter.** Ported reference pages should compile against the adapter without changes — just switch `react-router-dom` to `next/navigation`, replace asset imports with public/-relative strings, and add `'use client';`.

7. **Pre-existing test-file errors are not our problem.** ~79 lines of vitest Mock typing errors in `CelebrationOverlay.test.tsx`, `useMobileDetect.test.ts`, `usePullToRefresh.test.ts`, `usePushNotifications.test.ts` exist on this branch and predate Phase 16 / the redesign. Leave them alone. Plan 1 acceptance doesn't require them to pass.

8. **Sonner toaster.** Reference uses `sonner` with `theme="dark" position="top-center"`. Task 2.2 may need to add it. Check `apps/web/package.json` first — if it's already a dep, just mount the `<Toaster>` in the root layout. If not, `pnpm --filter @eureka-lab/web add sonner`.

9. **Don't push without user consent each time.** The first push (after Phase 0) was explicitly authorized by the user. Subsequent pushes — assume the PR auto-updates when you push, but ask before pushing in case the user wants a checkpoint.

10. **The `(game)` route group should be GONE post-revert.** If `apps/web/src/app/(game)/` exists, the revert didn't catch everything — investigate before deleting (might be from a stash apply or a manual edit).
